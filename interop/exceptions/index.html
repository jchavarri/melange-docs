Exceptions

  1. Usage

In the JS world, exception could be any data, while an OCaml exception is a structured data format and supports pattern matching. Catching an OCaml exception on JS side therefore doesn't work as intended.

JS exceptions can be raised from the Melange side by using the JS.Exn.raise* functions, and can be caught as a Melange exception of the type Js.Exn.Error with the JS exception as its payload, typed as Js.Exn.t. The JS Exception can then either be manipulated with the accessor functions in Js.Exn, or casted to a more appropriate type.

mlrejs 
let () = try
  Js.Exn.raiseError "oops!"
with
| Js.Exn.Error e ->
  match Js.Exn.message e with
  | Some message -> Js.log {j|Error: $message|j}
  | None -> Js.log "An unknown error occurred"
let () =
  try(Js.Exn.raiseError("oops!")) {
  | Js.Exn.Error(e) =>
    switch (Js.Exn.message(e)) {
    | Some(message) => Js.log({j|Error: $message|j})
    | None => Js.log("An unknown error occurred")
    }
  };

'use strict';

var Js_exn = require("bs-platform/lib/js/js_exn.js");
var Caml_js_exceptions = require("bs-platform/lib/js/caml_js_exceptions.js");

try {
  Js_exn.raiseError("oops!");
}
catch (raw_e){
  var e = Caml_js_exceptions.internalToOCamlException(raw_e);
  if (e.RE_EXN_ID === Js_exn.$$Error) {
    var message = e._1.message;
    if (message !== undefined) {
      console.log("Error: " + message);
    } else {
      console.log("An unknown error occurred");
    }
  } else {
    throw e;
  }
}
